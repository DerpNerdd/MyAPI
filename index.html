Rhythm Games API Documentation

Your guide to using the Rhythm Games and Songs API.

Introduction

Welcome to the Rhythm Games API! This API allows you to access a collection of rhythm games and their associated songs. You can retrieve lists of games, search for specific games or songs, and even manage the database using our admin endpoints. Below is a comprehensive guide to all available API endpoints.

Base URL

All requests should be made to the following base URL:

http://localhost:3000/api

Endpoints

1. Get All Rhythm Games

Retrieve a list of all rhythm games in the database.

Endpoint: GET /rhythmgames

Example Request:

GET http://localhost:3000/api/rhythmgames

Optional Query Parameters:

  • search: Search games by name (e.g., ?search=Sound)
  • limit: Limit the number of results (e.g., ?limit=10)

2. Get a Single Rhythm Game

Retrieve details of a specific rhythm game by its ID.

Endpoint: GET /rhythmgames/:id

Example Request:

GET http://localhost:3000/api/rhythmgames/1

3. Get All Songs

Retrieve a list of all songs in the database.

Endpoint: GET /songs

Example Request:

GET http://localhost:3000/api/songs

Optional Query Parameters:

  • search: Search songs by title (e.g., ?search=ALBIDA)
  • limit: Limit the number of results (e.g., ?limit=5)

4. Get a Single Song

Retrieve details of a specific song by its ID.

Endpoint: GET /songs/:id

Example Request:

GET http://localhost:3000/api/songs/1

Admin Endpoints

To perform admin actions (adding, editing, or deleting rhythm games and songs), you need to include the admin token in the URL. The token is used to authenticate admin requests.

Admin Token: admin123

5. Add a New Rhythm Game

Description: Add a new rhythm game to the database by specifying the game name, company, and admin token directly in the URL.

Endpoint: GET /api/rhythmgames/add/:name/:company/:token

Example Request:

http://localhost:3000/api/rhythmgames/add/GameName/Company/admin123

6. Add a New Song

Description: Add a new song to the database by specifying the song title, associated game ID, song level, and admin token directly in the URL.

Endpoint: GET /api/songs/add/:title/:game_id/:song_level/:token

Example Request:

http://localhost:3000/api/songs/add/SongTitle/1/20/admin123

7. Update a Rhythm Game

Description: Update an existing rhythm game by its ID, providing the new name, new company, and admin token directly in the URL.

Endpoint: GET /api/rhythmgames/:id/update/:name/:company/:token

Example Request:

http://localhost:3000/api/rhythmgames/1/update/NewGameName/NewCompany/admin123

8. Update a Song

Description: Update an existing song by its ID, providing the new song title, new game ID, new song level, and admin token directly in the URL.

Endpoint: GET /api/songs/:id/update/:title/:game_id/:song_level/:token

Example Request:

http://localhost:3000/api/songs/1/update/NewSongTitle/1/22/admin123

9. Delete a Rhythm Game

Description: Delete a rhythm game from the database by its ID, using the admin token directly in the URL.

Endpoint: GET /api/rhythmgames/:id/delete/:token

Example Request:

http://localhost:3000/api/rhythmgames/1/delete/admin123

10. Delete a Song

Description: Delete a song from the database by its ID, using the admin token directly in the URL.

Endpoint: GET /api/songs/:id/delete/:token

Example Request:

http://localhost:3000/api/songs/1/delete/admin123
